---
title: 'Create Zone'
api: 'POST /zone'
description: 'Create a new zone'
---

This endpoint creates a new zone for the authenticated user's account.

### Body

<ParamField body="name" type="string" required>
  The name of the zone to be created.
</ParamField>

### Response

<ResponseField name="zone" type="object">
  The created zone object.
</ResponseField>

<RequestExample>
```bash Example Request
curl -X POST 'https://api.yourdomain.com/zone' \
-H 'Content-Type: application/json' \
-H 'Authorization: Bearer YOUR_API_KEY' \
-d '{"name": "My New Zone"}'
```
</RequestExample>

<ResponseExample>
```
json Response
{
  "zone": {
    "id": "60a3e5b9f3f3a2001f3e8d1a",
    "name": "My New Zone",
    "account": "60a3e5b9f3f3a2001f3e8d1b",
    "createdBy": "60a3e5b9f3f3a2001f3e8d1c",
    "createdAt": "2023-05-18T12:00:00.000Z",
    "updatedAt": "2023-05-18T12:00:00.000Z"
  }
}
```
</ResponseExample>

### Error Responses

<ResponseField name="error" type="string">
  Error message if the zone creation fails.
</ResponseField>

- HTTP 403: Returned if the user doesn't have the required subscription or has reached the zone limit.
- HTTP 409: Returned if a zone with the same name already exists.
- HTTP 500: Returned if there's a server error during zone creation.
