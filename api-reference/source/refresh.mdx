---
title: 'Refresh Source'
api: 'PUT /source'
description: 'Refresh a source'
authMethod: "bearer"
---

This endpoint refreshes a source for the authenticated user's account, updating its associated vectors.

### Request Body

<ParamField body="_id" type="string" required>
  The ID of the source to be refreshed.
</ParamField>

### Response

<ResponseField name="message" type="string">
  A success message indicating the refresh was successful.
</ResponseField>

<RequestExample>
```bash
curl -X PUT 'https://api.privvy.xyz/source' \
-H 'Authorization: Bearer YOUR_API_KEY' \
-H 'Content-Type: application/json' \
-d '{"_id": "60a3e5b9f3f3a2001f3e8d1a"}'
```
</RequestExample>

<ResponseExample>
```json
{
  "message": "Refresh successful"
}
```
</ResponseExample>

### Error Responses

<ResponseField name="error" type="string">
  Error message if the source refresh fails.
</ResponseField>

- HTTP 400: Returned if the source upload is currently in progress.
- HTTP 404: Returned if the source is not found.
- HTTP 500: Returned if there's a server error during source refresh.

<Note>
  This operation updates the vectors associated with the source. It cannot be performed if the source is currently being uploaded.
</Note>
